Four stages. At each one you prompt IBM Bob to do the real work — build the server, containerize it, register it, grant it, drive the controls — while make handles the plumbing. Copy the prompts straight from here into Bob. Every stage has a one-command fallback if a live edit wobbles.

Architecture at a glance

IBM Bob drives a fintech (“FinByte”) agent mesh, but it never touches the backends directly. Every tool call and every agent-to-agent call flows through IBM ContextForge — the one governed seam that authenticates, authorizes, redacts, and audits before Bob sees a result and before any money moves. An OPA sidecar decides policy; the four controls are enforced right here.

No Docker on your laptop? Drive the whole mesh with just Bob

You don't need Docker, uv, or make to drive this. If the governed stack is running somewhere — a teammate's machine, a VM, or a GitHub Codespace — your laptop only needs IBM Bob. One command points Bob at the gateway over HTTPS:

make connect

It prints a ready-to-paste bob mcp add finbyte-gateway … -t http line (streamable-HTTP on /mcp — verified through the Codespaces proxy). Redaction, least-privilege, and OPA all still apply over that remote connection. See docs/ONBOARDING.md.

1 Bob builds an MCP server from scratch ungoverned · :8000

The starting point every developer knows: a plain MCP server on your laptop. No gateway, no policy, no auth — whoever reaches the port runs any tool. That exposure is the problem Stage 2 solves.

① Tell Bob to build it
Create a new MCP server with fastmcp at mcp-servers/sales-tax/server.py: a tool add_tax(amount, rate_pct=8.5) returning {amount, rate_pct, tax, total}, plus a GET /health route, served over HTTP on 0.0.0.0:8000.

Bob writes, runs, and fixes its own code — if the first server.py won't start, paste the error back and Bob iterates until it runs.

What Bob produced (mcp-servers/sales-tax/server.py) 
from fastmcp import FastMCP

mcp = FastMCP("sales-tax")

@mcp.tool
def add_tax(amount: float, rate_pct: float = 8.5) -> dict:
    """Add sales tax to an amount. `rate_pct` is a percentage, e.g. 8.5 for 8.5%."""
    tax = round(amount * rate_pct / 100, 2)
    return {
        "amount": amount,
        "rate_pct": rate_pct,
        "tax": tax,
        "total": round(amount + tax, 2),
    }

@mcp.custom_route("/health", methods=["GET"])
async def health(request):
    from starlette.responses import JSONResponse
    return JSONResponse({"status": "ok", "server": "sales-tax"})

if __name__ == "__main__":
    mcp.run(transport="http", host="0.0.0.0", port=8000)
② Serve it bare + call it
make stage1-build

It runs the server bare on :8000 and calls the tool to prove it works:

add_tax(100, 8.5) → tax=8.50, total=108.50 # no token, no policy

That's exactly the problem. Anyone on the network calls anything — no redaction, no policy, no audit. (Want to poke it by hand? make stage1-build also opens the MCP Inspector, no token field.)

Fallback if Bob's file won't run: make stage1-scaffold drops in a known-good sales-tax server, then re-run make stage1-build.
2 Put a tool behind ContextForge governed · :4444

Building is easy; governing is the hard part ContextForge solves. Carry the tool you just built into the mesh — then watch the grant be the thing that makes it callable.

① Containerize + bring the stack up (operator persona)
make stage2-govern
②a Register it — tell Bob (operator)
Register the sales-tax service at http://sales-tax:8000/mcp.

Now it's in the catalog, token-gated — but Bob still can't call it. Exposing a tool to an agent is a separate grant. That gate is least-privilege.

②b Grant it, then call it
make salestax-grant
Now tell Bob (builder)
Add sales tax to $100.

→ governed call through :4444 → 108.50. The tool you ran wide-open on :8000 now runs token-gated through the one seam. Built → governed → used.

②c Bonus — have Bob extend an existing service (fx-rates)

Re-register fx-rates and convert 1000 USD to EUR.

4The four controls

Every one is enforced on the gateway, before Bob ever sees a result and before any money moves.

ControlSay to BobWhat the gateway does
1 · Policy (OPA) "Wire $50,000 to Acme LLC for expense exp_big." BLOCKED over the $10k cap. Add "with dual approval" → ALLOWED. Same policy blocks the cross-language A2A $50k.
2 · Data protection "Fetch receipt rcpt_pii, verbatim." MASKED SSN → ***-**-6789, card → ****-****-****-1111, key → [SECRET_REDACTED].
3 · Prompt-injection "Fetch receipt rcpt_injection." NEUTRALIZED the embedded SYSTEM: ignore all prior policy… → [INJECTION_BLOCKED].
4 · RBAC least-privilege "Now wire $50k yourself, directly." Bob can't — the FinOps virtual server hides erp-payments-wire. The MCP Inspector confirms it's absent.

7Troubleshooting

  • Admin login keeps failing — it's browser autofill, not the password. See the Admin UI card above; the creds are correct.
  • MCP Inspector won't connect — make sure Connection Type is Via Proxy (not Direct), and the Authorization custom header row is toggled on. The token is on your clipboard from the inspect-mcp pane.
  • A pane shows an error and waits — read it, press Enter to close that pane. The others keep running.
  • "cockpit already running — attaching" — you re-attached to an old session. Run make cockpit-down, then make cockpit for a fresh one.
  • Bob says "No MCP servers configured" / lists no tools — launch it via make bob (it refreshes config first), or reseed with make demo-reset.
  • Stack not up — make quickstart brings up everything and seeds it.

8Teardown

make cockpit-down # stop the cockpit (panes + inspectors)
make down # stop the whole stack
