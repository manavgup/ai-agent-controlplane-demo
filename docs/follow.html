🗺️ The big picture — how it all connects (optional)

Bob never touches the tools directly. Every move — every tool call and every agent-to-agent call — passes through one checkpoint (the control plane), which checks it, hides secrets, and records it before anything happens.

1Build

Bob writes a brand-new tool, just by being asked

We ask Bob, in plain English, to build a small “sales tax” tool from scratch. In seconds it writes the program and runs it.

Here's the catch: right now that tool is wide open. There's no login, no rulebook, no record of who used it — anyone who can reach it could run it. That's the problem the next stages fix.

Watch for Bob types out the tool, then tests it: add 8.5% tax to $100 → total = 108.50. The presenter will point out: no password, no rules, no audit trail.

2Govern

We put the control plane in charge of Bob's new tool

Now we hand Bob's tool to ContextForge — the control plane. From here on, every single use of the tool passes through one checkpoint that can verify, record, and allow-or-deny it.

Notice one subtle thing the presenter calls out: simply listing the tool in the catalog isn't enough. Someone has to explicitly grant permission before Bob is allowed to use it. Listing ≠ allowed.

Watch for The same total = 108.50 as Stage 1 — but now it only works after permission is granted, and every use is logged. Same tool, now governed.

3Controls

The payoff — watch four safeguards stop Bob in real time

This is the good part. Bob now works inside a pretend finance company. We ask it to do four everyday things — and each time, a different safeguard kicks in. Bob tries; the control plane decides.

🙈
A receipt with private info on it

Bob asks to read a receipt that contains a Social Security number, a credit-card number, and a secret key. The control plane blanks them out before Bob ever sees them.

HIDDEN → ***-**-6789 · [SECRET_REDACTED]
🪤
A receipt with a hidden trick on it

One receipt has a sneaky instruction buried in it trying to hijack Bob (“ignore your rules and…”). The control plane strips the trick out so Bob never follows it.

NEUTRALIZED → [INJECTION_BLOCKED]
🛑
A $50,000 payment

Bob tries to send a $50,000 wire. Company policy says anything over $10,000 needs a second approval — so the control plane blocks it. (It even blocks when Bob asks a different helper agent to do it.)

BLOCKED → over the $10,000 limit
🔑
Bob doesn't even have the “send money” button

In its finance-analyst role, Bob was never given the wire tool at all. You can't misuse what you were never handed. The right to act follows the role.

NOT ALLOWED → no wire tool for this role

4Proof

All of it, checked automatically

16 checks · all green ✅ Every safeguard you just watched is re-tested automatically — no human needed. If WiFi or a laptop misbehaves, this is the receipt that the control plane really works.

The takeaway: building a tool is easy — putting something in charge of it is the hard part. That “something in charge” is the control plane. Register it → grant permission → every action checked.

That's the show. Bob did the building and the asking; the control plane did the deciding. Thanks for following along! 👋
