Drive Bob yourself

Install only IBM Bob. Point it at the control plane running in the cloud and watch the AI get governed.

The whole governed mesh runs in the cloud — a GitHub Codespace the presenter started. You install only IBM Bob on your laptop (no Docker), connect it to that gateway over the wire, and drive the controls live. Every prompt you type is checked by the same control plane everyone else is using.

1Install IBM Bob

One command on macOS/Linux. Needs Node ≥ 22.15 (it checks for you).

curl -fsSL https://bob.ibm.com/download/bobshell.sh | bash

Prefer a guide? Install guide ↗ · Download ↗. Then sign in with bob.

2Get your connect line — copy or download, don't type it

The connection carries a ~470-char token. Typing it is hopeless, so the dashboard hands it to you. Open the same dashboard link the presenter shared and click 🔌 Connect Bob (laptop) — then pick one:

  • Copy the command — one click copies the full bob mcp add … line (token and all). Paste it into your terminal.
  • Or one fixed line — same for everyone, no token to copy. Run it in an empty folder; it drops the config in and launches Bob:
mkdir -p .bob && curl -fsSL <dashboard-url>/bob/settings.json -o .bob/settings.json && bob
Gotchas: use -t http (never -t sse) and run Bob from an empty folder (not a clone of the repo — its .bob/mcp.json would shadow this). The Connect Bob page guarantees the full token, so no more silent truncation.

Presenter setup: start the companion with EXPOSE_CONNECT=1 make companion (it's off by default), then share the dashboard link. Running your own Codespace? Open in Codespaces ↗, set port 4444 → Public. Bob is not installed in the Codespace — it's the mesh host; you drive from your laptop.

3Connect Bob

Paste the line into your terminal (or add it in Bob's MCP manager UI).

bob mcp list
You'll see finbyte-gateway connected. A persona is just which gateway URL you point Bob at — same gateway, same token, swap the URL → swap the toolset. That's RBAC, made of one URL.

5Register your own agent Operator — optional

Join the room's live count. Switch Bob to the Operator persona first — on the 🔌 Connect Bob page, copy the Operator command (or download its settings.json) and re-add. Then:

Register an MCP server named salestax-<YOUR-INITIALS> at http://sales-tax:8000/mcp?agent=<YOUR-INITIALS>.
You'll see “Agents built by the room” climb on every phone and the projected wall. The ?agent=… suffix keeps each agent's URL unique (ContextForge requires it; the backend ignores the query).
🚀 Going further — register a tool you built on your own laptop
Have Bob write a real MCP server locally, expose it with a one-line tunnel, then register that public URL with the central gateway:
cloudflared tunnel --url http://localhost:8000
Register the printed https://<random>.trycloudflare.com/mcp as your agent's URL. The gateway dials your laptop back through the tunnel and governs every call — proven end to end (a call routed to a laptop returned total 108.50 through the cloud gateway). Throwaway demo data only; the tunnel is public.
