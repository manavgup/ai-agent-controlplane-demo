How to test it yourself

  1. Bring up the stack and seed it (includes the 5 voter agents): 
    make up && make seed
docker compose up -d --build room-agent   # if not already running
    On Podman/macOS, prefix with DOCKER_BUILDKIT=0. If a base-image pull fails with an auth error, run docker logout docker.io and retry.
  2. Prove the controls headlessly (no browser): 
    make verify-controls     # → RESULT: 19 passed, 0 failed
make verify-quorum       # → quorum: 3 passed, 0 failed
  3. Unit-test the pure voting logic: 
    cd a2a-agents/room && uv run --with pytest pytest test_vote.py -q   # → 11 passed
  4. Drive it live: 
    make companion
    then open http://localhost:7070. Type initials, tap Approve/Reject, then click Run on the "Expense approval quorum" card → BLOCKED with the vote breakdown. Click Evidence for the OPA log. Open /wall on a second screen. There are two quorum cards: the Companion-driven one and "Agent-orchestrated quorum — chair agent (A2A)" where a real agent runs it.
  5. See the raw A2A protocol (agent cards + the JSON-RPC Task/Artifact envelope): 
    make inspect-a2a
    opens the official a2a-inspector; point it at the chair (:9200) or a voter (:9100) to fetch its agent card and send a live message.
  6. Test from a phone / a room: 
    make present
    opens public tunnels and your browser to the join QR. Scan it from a phone and vote. For a public room set PRESENTER_KEY=… so attendees can't freeze the vote (you open the dashboard at /?k=<key>).
  7. Poke the API directly: 
    curl -s -X POST "localhost:7070/api/vote?initials=ME&choice=approve"
curl -s localhost:7070/api/run/quorum | python3 -m json.tool   # verdict: BLOCKED
