🔍 Verify what's registered with the gateway (Admin UI · MCP & A2A inspectors)

Your Codespace already ran make up && make seed, so the mesh is up. Run these in the Codespace terminal (the gateway is localhost there), then open the matching forwarded port from the PORTS tab — 4444 / 6274 / 8090 are pre-forwarded by the devcontainer.

① Admin UI — the fastest “is it registered?” check start here

make monitor # opens the Admin UI → /admin on :4444
You'll see the ContextForge Admin UI → Catalog: every registered MCP server, tool, virtual server, and A2A agent in one list. No client needed.

② MCP Inspector — the governed tool slice a client actually gets

make inspect-mcp
You'll see the Inspector on :6274 → 8 tools, with erp-payments-wire absent — least-privilege, proven from the client side.
In a Codespace? The Inspector's UI runs in your laptop browser, so its proxy (port :6277) must be reachable and must trust the forwarded UI origin. make inspect-mcp now sets both for you — MCP_PROXY_FULL_ADDRESS (where the proxy is) and ALLOWED_ORIGINS (whitelists the …-6274.app.github.dev origin, else the proxy blocks it with a “DNS-rebinding / invalid origin” error). You only: set port 6277 → Public, open the printed URL, fill Transport = Streamable HTTP, the localhost:4444/servers/…/mcp URL, Via Proxy, the Authorization header. No-fuss alternative: surface ①, the Admin UI.

③ A2A Inspector — validate the two agent cards

make inspect-a2a
You'll see the A2A Inspector on :8090 → the Python auditor + Rust payments agent cards (the cross-language pair Stage ③'s $50k block fires across).
Driving from your laptop? Use surface ① (open the :4444/admin page) — it needs nothing installed. To point a local MCP Inspector at the public gateway instead: npx -y @modelcontextprotocol/inspector → Transport Streamable HTTP, URL https://<codespace>-4444.app.github.dev/servers/<FinOps-UUID>/mcp, Via Proxy, add header Authorization: Bearer <token>. If the Inspector asks for a proxy token, open the pre-filled URL it prints.
Architecture — what you're building (optional)

Bob drives a fintech (“FinByte”) agent mesh, but never touches the backends directly. Every tool call and every agent-to-agent call flows through ContextForge — the one governed seam that authenticates, authorizes, redacts, and audits before Bob sees a result or any money moves. An OPA sidecar decides policy.

Where it runs — how the whole room connects (the single control plane)

The control plane is one Docker Compose stack on one host — gateway + OPA + 6 MCP servers + 2 A2A agents, plus the Companion dashboard. A GitHub Codespace is the recommended single instance (the devcontainer runs make up && make seed; you add make companion). Only two ports are exposed to attendees — and all three follow-along tiers connect through them.

1Build — Bob writes an MCP server from scratch ungoverned · :8000expand/collapse

The starting point every developer knows: a plain MCP server on your laptop. No gateway, no policy, no auth — whoever reaches the port runs any tool. That exposure is the problem Stage ② fixes.

Self-check: Stage ① succeeded if

2Govern — register → grant → call governed · :4444expand/collapse

Building is easy; governing is the hard part ContextForge solves. Carry the tool you built into the mesh, then watch the throughline: registering only catalogs it — granting is what makes it callable.

Self-check: Stage ② succeeded if
  • make stage2-govern reports the sales-tax container healthy and switches Bob to the Operator persona.
  • Bob registers sales-tax and the result includes "registered":true, "status_code":200, and "reachable":true.
  • After make salestax-grant, Bob can call sales-tax-add-tax and gets total=108.5 through the gateway.
1 ⌨ Run in your terminal — containerize + bring the stack up
make stage2-govern
You'll see your sales-tax tool containerized onto the mesh and the gateway come up. Bob is switched to the Operator hat.
🔌 How does Bob connect to the gateway? (the “hat” is just an MCP connection)

“Switched to the Operator hat” isn't magic — make bob-operator just writes Bob's MCP server connection. Here's the real thing — this is exactly what make connect prints, filled in:

bob mcp add finbyte-gateway "http://localhost:4444/servers/<vserver-UUID>/mcp" -t http -H "Authorization: Bearer <token>" --trust

…or add it in Bob's UI (the MCP manager). Inspect your connections with bob mcp list.

The whole trick: a persona is just which gateway virtual-server UUID you point Bob at — Analyst, Operator, or Builder. Same gateway, same token; swap the UUID → swap the toolset. That's RBAC, made of one URL.
