Architecture — what you're building (optional)

Bob drives a fintech (“FinByte”) agent mesh, but never touches the backends directly. Every tool call and every agent-to-agent call flows through ContextForge — the one governed seam that authenticates, authorizes, redacts, and audits before Bob sees a result or any money moves. An OPA sidecar decides policy.

1Build — Bob writes an MCP server from scratch ungoverned · :8000

The starting point every developer knows: a plain MCP server on your laptop. No gateway, no policy, no auth — whoever reaches the port runs any tool. That exposure is the problem Stage ② fixes.

1Builder 🗣 Say this to Bob
Create a new MCP server with fastmcp at mcp-servers/sales-tax/server.py: a tool add_tax(amount, rate_pct=8.5) returning {amount, rate_pct, tax, total}, plus a GET /health route, served over HTTP on 0.0.0.0:8000.
You'll see Bob write mcp-servers/sales-tax/server.py — a working FastMCP server. If it won't start, paste the error back and Bob fixes its own code.

2Govern — register → grant → call governed · :4444

Building is easy; governing is the hard part ContextForge solves. Carry the tool you built into the mesh, then watch the throughline: registering only catalogs it — granting is what makes it callable.

1 ⌨ Run in your terminal — containerize + bring the stack up
make stage2-govern
You'll see your sales-tax tool containerized onto the mesh and the gateway come up. Bob is switched to the Operator hat.
🔌 How does Bob connect to the gateway? (the “hat” is just an MCP connection)

“Switched to the Operator hat” isn't magic — make bob-operator just writes Bob's MCP server connection. Here's the real thing — this is exactly what make connect prints, filled in:

bob mcp add finbyte-gateway "http://localhost:4444/servers/<vserver-UUID>/mcp" -t http -H "Authorization: Bearer <token>" --trust

…or add it in Bob's UI (the MCP manager). Inspect your connections with bob mcp list.

The whole trick: a persona is just which gateway virtual-server UUID you point Bob at — Analyst, Operator, or Builder. Same gateway, same token; swap the UUID → swap the toolset. That's RBAC, made of one URL.
2Operator 🗣 Say this to Bob — register it
Register the sales-tax service at http://sales-tax:8000/mcp.
You'll see Bob call register-mcp-server → sales-tax appears in the catalog, token-gated — but Bob still can't call it. Granting is a separate step. That gate is least-privilege.

3Control — the four guardrails fire the payoff

Being in the catalog buys auth + one seam. The interesting controls live in the FinOps mesh — watch each one bite a real call Bob makes. This is the part a connected-only Bob (option ②) can drive live.

1 ⌨ Run in your terminal — seed the FinOps mesh
make stage3-controls
You'll see the FinOps mesh seeded and Bob switched to the Analyst hat (8 tools, no wire).
2 ⌨ Run in a second pane — watch the decisions
make logs-opa
You'll see a live ALLOW / DENY stream — every policy decision with its reason, in real time.
3Analyst 🗣 Say this to Bob — one prompt hits all four controls
Process today's pending expenses end to end. For each: read it, check its receipt, approve and reimburse anything clean, and flag anything that needs a wire.
